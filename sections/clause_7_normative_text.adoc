=== Emission Event Class

An Emission Event represents the release of a substance into the atmosphere over a non-zero duration. It is characterized by its temporal bounds (start time and end time), the Source Feature from which the substance originated, and the identity and quantity of the emitted substance. Both start time and end time are mandatory; each carries a Temporal Status that explicitly indicates whether the time instant is known, unknown, or ongoing.

NOTE: Conceptually, an Emission Event corresponds to an occurrent in upper-ontology terms, as it unfolds over time and has temporal parts.

NOTE: For reporting or accounting purposes, a long-running emission event is often segmented into multiple time-bounded emission events (e.g., annual records). This segmentation reflects reporting conventions and does not imply physical cessation or re-initiation of the emission.

|endTime | The end time of the emission event. The `status` attribute of the TemporalBound indicates whether the end time is known, unknown, or ongoing. | TemporalBound |1 (mandatory)
|hasQuantity | The quantity of the substance emitted. | EmissionQuantity |1 (mandatory)
|hasStartTimeBasis | The procedure used to determine the start time. | DeterminationProcedure |1 (mandatory)
|hasEndTimeBasis | The procedure used to determine the end time. SHALL be present when `endTime.status = known` (Constraint C4). | DeterminationProcedure |1 (mandatory)

*Constraint C4*: If `endTime.status = known`, `hasEndTimeBasis` SHALL be present. The value MAY be "unknown" or "legacy" when the method is not documented.

*Constraint C5*: If `endTime.status != known` (i.e., `unknown` or `ongoing`), `hasEndTimeBasis` and `hasEndTimeEvidence` are not applicable and SHOULD be omitted.

NOTE: `startTime.status = unknown` arises when the actual emission onset is undetermined — for example, in inventory-based emissions where annual totals are back-calculated from fuel consumption or production records, or in legacy data where the start time was never recorded. In most operational contexts, `startTime.status = known` is expected (see Constraint C3).

=== Feature Class

A Feature is an abstraction of real world phenomena (Source: ISO 19101-1:2014). In EmissionML, Feature serves as the base class for entities that can be the ultimate feature of interest of an Observation. A Feature's `geometry` attribute provides the geospatial reference for the phenomenon it represents.

=== Source Class

A Source represents the feature from which an emission originates. A Source is an ISO 191xx feature whose geometry provides the geospatial reference for one or more Emission Events. A Source is characterised by its own identity, geometry, intended function, and lifecycle, and is explicitly agnostic to the Mechanism by which any associated Emission Event occurs.

NOTE: A Source is a continuant that may exist independently of any emission, whereas an Emission Event is an occurrent. The identity and lifecycle of a Source are independent of the existence, absence, or frequency of Emission Events.

=== Temporal Bound Class

A Temporal Bound is a value object representing a temporal determination (e.g., start time or end time) whose `status` attribute explicitly indicates whether the time instant is known, unknown, or ongoing. This eliminates ambiguity that arises when absence of a value is used to signal multiple distinct states. A Temporal Bound MAY have associated data quality metadata. The determination procedure and supporting evidence for a Temporal Bound are tracked at the Emission Event level.

[width="90%",cols="2,6"]

Each Temporal Bound class shall have the mandatory attributes listed in <<temporal-bound-attributes>>.

=== Observation Class

An Observation is an act carried out by an observer to determine the value of an observable property of an object (feature-of-interest) by using a procedure, with the value provided as the result (Source: ISO/OGC 19156:2023). In EmissionML, an Observation is evidence that may support an Emission Quantity or Temporal Bound determination.

NOTE: The Ultimate Feature of Interest of an Observation identifies a Feature — which may be a Source or another feature at a different spatial scale — associated with the Emission Event. It is common for a single Emission Event to be observed by multiple Observations whose Ultimate Features of Interest are defined at different spatial scales. For example, a satellite remote sensing image Observation typically has an area-based Ultimate Feature of Interest (e.g., a facility area or region), whereas the same Emission Event may also be observed by a person carrying an Optical Gas Imaging camera, for which the Ultimate Feature of Interest is a specific Source such as a component of an item of equipment from which pollutants are released.
